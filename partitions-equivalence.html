Partitions d’équivalence

Diviser intelligemment les données pour un testing efficace.

Définition

Les partitions d’équivalence divisent les données en partitions (ou ensembles) dans l’idée que toutes les valeurs appartenant à une même partition sont traitées de manière similaire par le système sous test.

L’idée est la suivante : si une valeur d’une partition déclenche un comportement particulier (correct ou incorrect), alors les autres valeurs de cette même partition auront le même effet. Ainsi, il suffit de tester une valeur représentative de chaque partition, plutôt que toutes les valeurs possibles.

Partitions valides et invalides

On distingue souvent :

  • Les partitions valides : valeurs attendues par le système et qui doivent être correctement prises en charge.
  • Les partitions invalides : valeurs imprévues ou hors du périmètre normal, qui doivent être soit rejetées, ignorées ou traitées différemment.

Tester également les partitions invalides permet de s’assurer que le logiciel gère convenablement les erreurs ou valeurs hors limites.

Notion de couverture

Dans la technique des partitions d’équivalence, le but est d’atteindre 100% de couverture en testant au moins une valeur de chaque partition identifiée. Cette couverture se mesure en comparant :

  • Nombre de partitions exercées (via les cas de test)
  • Nombre total de partitions identifiées

Ainsi, la couverture exprimée en pourcentage est : (partitions testées / partitions totales) × 100.

Couverture “Each Choice”

Pour les systèmes avec plusieurs ensembles de partitions (par exemple, plusieurs paramètres d’entrée), la stratégie la plus courante s’appelle Each Choice. Elle exige que chaque partition de chaque ensemble soit testée au moins une fois.

Par exemple, si vous avez 2 paramètres d’entrée, chacun divisé en 3 partitions, vous devez au minimum créer des tests qui couvrent les 3 partitions du paramètre A et les 3 partitions du paramètre B. Cependant, Each Choice ne garantit pas la couverture de toutes les combinaisons possibles.

Exemple illustratif

Imaginons un champ de saisie d’âge (0 ≤ age ≤ 120) :

  • Partition invalide 1 : age < 0
  • Partition valide : 0 ≤ age ≤ 120
  • Partition invalide 2 : age > 120

On peut alors tester -1 pour la partition “age < 0”, 30 pour la partition “0 ≤ age ≤ 120”, et 200 pour la partition “age > 120”. Ainsi, on couvre les 3 partitions identifiées.

Exercice interactif

Saisissez un intervalle valide (min et max) pour le champ d’âge. Le script affichera ensuite les partitions invalide / valide / invalide, ainsi que des valeurs de test suggérées.

Conclusion

Les partitions d’équivalence offrent un moyen efficace de réduire le nombre de tests nécessaires, tout en maintenant une couverture satisfaisante. Elles restent toutefois plus puissantes quand on les combine avec d’autres techniques, comme l’analyse des valeurs limites ou encore les tests de transition d’état.
